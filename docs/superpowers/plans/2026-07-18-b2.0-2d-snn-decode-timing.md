## Task 1: Round 1 — disaccoppiare readout ↔ decode (+1 registro)

**Files:**
- Modify: `matlab/build_hdl_variants.m` (funzione locale `acciidm_m_chart_code`, blocco `Lmain`)

Oggi: `if valid { rawl=raw; phase=1 }` è **prima** della catena `if phase==1 {decode} elseif...`. Nel ciclo `valid`, `phase` è messo a 1 e la catena lo rilegge subito → decode nello stesso ciclo del readout. Spostando il blocco `if valid` **dopo** la catena, nel ciclo `valid` la catena (phase==1..7) è saltata (phase è 0 durante il run SNN) e solo il latch scatta; il decode va al ciclo dopo → `rawl` diventa un vero registro → il path si spezza in `max(readout, decode)`.

- [ ] **Step 1: Spostare il blocco `if valid ... end` dopo la catena delle fasi**

```matlab

    '    rawl(:) = raw;                     % LATCH e basta: il decode e'' il ciclo dopo'
    '    phase = uint8(1);'
    '  end'
```
e **reinserirlo** subito prima di `'  accel = acc;'` (dopo `'  end'` che chiude `elseif phase == 7`), con commento aggiornato:
```matlab
    '  if valid                             % [2d R1] latch DOPO la catena: il decode e'' il ciclo dopo'
    '    rawl(:) = raw;                      %   -> rawl e'' un vero registro -> readout e decode disaccoppiati'
    '    phase = uint8(1);'
    '  end'
```
Risultato: `phase==1` (decode) legge `rawl` registrato dal ciclo precedente; il readout `pR_idx→raw` e il decode `rawl→pv` sono path separati.

## Task 2: Round 2 — accorciare l'accumulo `reci` (16 termini)

**Files:**
- Modify: `matlab/snn_b2_fsm.m` (lo stadio C, il loop `reci`) — **oppure** una sotto-funzione fase, secondo la leva scelta.

Il `critpath.rpt` di R1 confermerà il collo (atteso: `reci` a 16 termini in ripple, ~130 CARRY4). Leva primaria = **ridurre la profondità dell'accumulo** mantenendo `dmax=0`. Due opzioni, scelte sui dati:

- [ ] **Step 1: Verificare se l'accumulo `reci` è a PRECISIONE PIENA (associativo → adder-tree bit-exact)**

Run (MATLAB): controlla che `T.accw` non saturi/arrotondi sui 16 termini (se pieno, sommare in ordine diverso è bit-exact).
```matlab
T = snn_types('fixed',13); nt = numerictype(T.accw); fprintf('accw: WL=%d FL=%d\n', nt.WordLength, nt.FractionLength);
```
Expected: se `accw` è abbastanza largo da contenere la somma dei 16 prodotti senza overflow/round, **l'adder-tree è bit-exact** (Step 2a). Altrimenti serve l'FSM-staging (Step 2b).

- [ ] **Step 2a: [se full-precision] Adder-tree bilanciato per `reci`** (profondità 16→4, bit-exact)

In `snn_b2_fsm.m`, sostituire il loop sequenziale
```matlab
    reci = cast(0, 'like', T.accw);
    for r = 1:rnk
      reci(:) = reci + cast(cast(W.U(i, r), 'like', T.w) * t_lr(r), 'like', T.accw);
    end
```
con una **riduzione ad albero** (somma a coppie, `log2(rnk)` livelli invece di `rnk`):
```matlab
    prod = cast(zeros(rnk,1), 'like', T.accw);
    for r = 1:rnk
      prod(r) = cast(cast(W.U(i, r), 'like', T.w) * t_lr(r), 'like', T.accw);
    end
    m = rnk;
    while m > 1
      h = floor(m/2);
      for q = 1:h, prod(q) = cast(prod(2*q-1) + prod(2*q), 'like', T.accw); end
      if mod(m,2)==1, prod(h+1) = prod(m); m = h+1; else, m = h; end
    end
    reci = prod(1);
```
(⚠️ bit-exact SOLO se `T.accw` è full-precision sui 16 termini — verificato Step 1. La somma ad albero cambia l'ORDINE, non il risultato, se non c'è arrotondamento intermedio.)

- [ ] **Step 2b: [se NON full-precision] FSM sub-stage dell'accumulo** (serializza `reci`/`Ii` su più cicli)

Se l'ordine conta (arrotondamento intermedio), non si può ribilanciare: si **spezza lo stadio C** su più cicli FSM (come le fasi IIDM), accumulando `reci` 1-2 termini/ciclo in un registro. Aggiunge cicli/neurone (irrilevante: time-mux). Dettaglio da instrumentare sul modello `snn_b2_fsm` mantenendo l'ordine di somma originale → bit-exact per costruzione (stesso ordine, solo registrato).
